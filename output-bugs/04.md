```
┌─────────────────────────────────────────────────────────────┐
│ CLIENT (OpenAI format)                                      │
│ messages: [                                                 │
│ {role: "user", content: "..."},                             │
│ {role: "assistant", tool_calls: [{id_123", ...}]}           │
│ {role: "tool", tool_call_id: "call_123", content: "..."}    │
│ ]                                                           │
└────────────────────────┬────────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────────────┐
│ PROXY GATEWAY                                               │
│                                                             │
│ 1. PARSE: Keep messages as structured array                 │
│ 2. ROUTE: Based on target provider                          │
│ 3. TRANSLATE (only if needed):                              │
│ • OpenAI→OpenAI: PASS THROUGH (no-op)                       │
│ • OpenAI→Anthropic: role="tool" → role="user"               │
│ + type="tool_result"                                        │
│ + tool_use_id = tool_call_id                                │
│ • OpenAI→Bedrock: role="tool" → {toolResult: {...}}         │
│ 4. STREAM: Accumulate tool_calls by index, keep             │
│ delta.content and delta.tool_calls SEPARATE                 │
│ 5. FORWARD: Send structured messages, never text            │
└────────────────────────┬────────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────────────┐
│ UPSTREAM LLM                                                │
│ Receives structured messages with proper role semantics     │
│ Knows tool outputs are NOT user narrative content           │
│ Generates assistant response without echoing tool output    │
└─────────────────────────────────────────────────────────────┘
```
