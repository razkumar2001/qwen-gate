# Qwen Gate — Agent Guide

## Start Here

At task start, call `memory_context` to surface related patterns and decisions.

## DeepWork

Use the deepwork skill for this task. Treat it as a heavy coding session.

Deepwork requirements:
- create/update a `.slim/deepwork/` progress file;
- keep OpenCode todos synced with the current phase;
- draft a plan and get `@oracle` review before implementation;
- create and review a phased implementation/delegation plan;
- execute phase by phase with background specialists where useful;
- wait for hook-driven background completion, reconcile results, validate, and ask `@oracle` to review each phase;
- ask `@oracle` to include simplify/readability feedback in phase reviews;
- fix actionable review issues before continuing.

## Memory

- **`memory_remember`** — store non-obvious findings. Scope = affected area (e.g. `streaming`, `strip-logic`). Call after discovering something an agent would otherwise rediscover.
- **`memory_recall`** — retrieve memories by scope/type/tag before starting related work.
- **`memory_context`** — call at task start to surface relevant patterns.
- **`memory_list`** — discover what scopes/types exist.
- **`memory_update`** / **`memory_forget`** — update or delete outdated memories.
- Tag memories with relevant keywords for targeted recall.

## Runtime

- **Bun only** (v1.3+). Node.js `start:node` script exists but is secondary.
- No build step: `bun src/index.tsx` runs TypeScript directly.
- Uses `bun.lock` (not `package-lock.json`).
- Import extensions use `.ts`/`.tsx` (see `allowImportingTsExtensions` in tsconfig).

## Commands

```bash
bun test              # All tests, no setup needed
bun start             # Dev server (`bun src/index.tsx`)
bun dev               # Hot reload (`bun --watch src/index.tsx`)
bun cluster           # Multi-core mode
bun qg                # CLI entry (`bun src/cli.ts`)
```

## Hard Rules

- **NEVER hardcode data that can be computed from known parameters.** If something depends on a keyword, prefix, or config value, generate it programmatically.
- **NEVER duplicate logic.** Extract shared code into a single source of truth (function, constant, or module) and import it everywhere.
- **Tag names** (tool call keywords, think tag names) live in `src/utils/tagNames.ts`. All consumers import from there.
- **Config values** (feature_config, Qwen API settings) live in `src/services/qwen.ts`. Export reusable builders, don't inline copies.

## Serena MCP

Serena v1.5.3 is installed with full toolset (36 tools). At session start, call `initial_instructions` before using its tools.

**Prefer Serena tools for all coding work.** Serena's tools are purpose-built for coding workflows -- symbol-aware, token-efficient, and more reliable than generic alternatives.

| Category | Serena tools | Prefer over OpenCode |
|---|---|---|
| Code reading | `read_file`, `get_symbols_overview`, `find_symbol`, `find_declaration`, `find_implementations` | `read` (only when you need raw file content) |
| Search | `search_for_pattern`, `find_referencing_symbols`, `find_file`, `list_dir` | `grep`, `glob` |
| Editing | `replace_symbol_body`, `rename_symbol`, `safe_delete_symbol`, `replace_content`, `delete_lines`, `replace_lines`, `insert_at_line`, `insert_after_symbol`, `insert_before_symbol` | `edit` (prefer Serena's symbolic/line tools) |
| File creation | `create_text_file` | `write` |
| Diagnostics | `get_diagnostics_for_file`, `get_diagnostics_for_symbol` | (no OpenCode equivalent) |
| Shell | `execute_shell_command` | `bash` |
| Project memory | `write_memory`, `read_memory`, `list_memories`, `delete_memory`, `rename_memory`, `edit_memory` | (no OpenCode equivalent) |
| Onboarding | `initial_instructions`, `onboarding`, `serena_info` | (no OpenCode equivalent) |

## Config

- `config.json` (not version controlled, generated by installer).
- All 23 config keys can be overridden by env vars of the same name (e.g., `PORT`, `API_KEY`). See `ConfigSchema` in `src/services/configService.ts` for the full key list. `STREAM_IDLE_TIMEOUT_MS` is env-only (not a config key).
